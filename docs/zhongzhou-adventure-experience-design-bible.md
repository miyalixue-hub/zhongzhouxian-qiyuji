## 1. 文档说明与版本记录

### 1.1 文档用途

本策划案用于建立《中轴奇游记——AI文物唤醒计划》的统一体验框架，把目前已有或计划中的地图探索、文物观察、纹饰与颜色选择、AI图像生成、3D模型和实体文创串联为同一个可理解、可参与、可延展的故事体验。

它主要回答以下问题：孩子在故事中是谁、为什么行动、每一步发现什么、AI扮演什么角色、文化知识如何进入操作，以及数字成果如何转化为可保存和可带走的作品。本文档不替代史料审定、页面原型、视觉规范、技术架构或生产工艺文件，而是为这些后续工作提供共同的体验依据。

### 1.2 使用对象

本文档面向：

- 产品策划、课程策划与项目负责人；
- 文化研究、内容编辑与史料审核人员；
- 交互、视觉、动画、声音和角色设计人员；
- 前端开发、AI功能、3D建模及打印制作人员；
- 博物馆、研学机构、学校教师及文创合作方。

不同岗位可据此拆分专业方案，但涉及玩家身份、角色关系、故事主线、文化口径和体验节奏的关键调整，应回到本文档统一记录。

### 1.3 文档维护方式

- 采用“分章节讨论—修改—确认”的方式递进，不一次性锁定全篇。
- 每轮只处理已约定范围；当前章节确认后，再进入下一阶段。
- 已确认内容如需改动，应记录改动原因、影响章节和确认人，避免局部修改破坏整体体验。
- 史料核验、角色设定、交互细节和技术实现中的未决事项，统一进入“待确认事项”，不以模糊措辞掩盖。
- 页面、课程、动画与文创方案落地后，应反向校验本文档；如实际实现发生变化，及时同步版本。

### 1.4 版本管理原则

版本号采用“主版本.次版本”的方式管理：

- `V0.x`：讨论与验证阶段，允许章节结构和关键设定调整；
- `V1.0`：核心世界观、角色、主线、体验旅程和首章方案全部确认，可作为研发基线；
- `V1.x`：不改变核心定位的增补、修订和落地细化；
- `V2.0`：核心体验、目标用户或产品形态发生重大变化。

每次更新须在版本记录中写明日期、范围、主要变化和状态。单次修订应围绕一个明确目标，便于产品、内容与研发追踪影响。

### 1.5 文化事实与故事创作的标注原则

为保护文化知识的可靠性，策划及后续页面内容采用三级标注：

| 标注 | 定义 | 使用要求 |
| --- | --- | --- |
| **【史实】** | 有考古报告、权威机构资料、正式出版物或可信研究支持的事实 | 记录来源；未经核验不写成确定事实 |
| **【文化解读】** | 基于器物形态、使用环境、传统观念或研究观点形成的合理解释 | 使用“可能”“可理解为”等边界清晰的表述，避免把寓意等同于唯一结论 |
| **【故事设定】** | 为儿童叙事、角色塑造和互动任务创作的拟人化内容 | 不冒充史实；必要时以“奇游记故事中”等方式明确提示 |

“守护兽失去记忆、颜色和力量”“孩子成为文物唤醒师”“AI帮助修复记忆”等属于【故事设定】。镇水兽、吸水兽的考古出处、位置、年代、用途及纹饰含义，在进入正式内容前须逐条完成来源核验。故事可以大胆想象，但不能替历史替孩子作答。

## 2. 项目背景与项目定位

### 2.1 项目背景

《中轴奇游记》属于“北京中轴线数字文创”项目的交互体验部分。项目以北京中轴线文化遗产为背景，从正阳门、正阳桥及其守护形象切入，面向儿童和青少年设计兼具探索、创作与成果转化的文化体验。

项目希望解决的不是“如何把更多资料放进网页”，而是“如何让孩子愿意靠近文物，并通过自己的行动理解它”。孩子从观察遗迹和器物开始，逐步辨认用途、纹饰、传统颜色与文化环境，再借助AI形成个性化2D形象和3D模型，最终延伸至打印模型与文创产品。

### 2.2 当前项目已有基础

根据当前仓库中的页面、技术架构文档与纹饰实现方案，项目已具备或已明确以下基础：

- 以手绘中轴线地图作为入口，设置9个站点，并以正阳门作为首个已解锁章节；
- 正阳门章节已有镇水兽、吸水兽和前门铛铛车等文化创作原型；
- 神兽创作流程已覆盖原型选择、纹饰选择、姿态与表情、传统色、附加元素、AI提示词确认、2D候选图选择、3D生成与下载；
- 已设置8类局部纹饰、8种颜色及多种文化附加元素；
- 已接入或规划AI图像生成、图生3D、模型预览、STL/3MF/GLB下载及打印工单；
- 已有“神兽档案”、保存分享、等待小游戏等成果与反馈机制；
- 纹饰参考图方案提出让孩子选择的纹饰真正参与AI生成，而非只停留在文字标签。

这些内容证明项目已经拥有较完整的“创作工具链”。后续策划的重点，是为这条工具链补上统一的行动动机、文化探索过程和情感结果。

### 2.3 当前存在的问题

1. **体验像连续配置，尚未形成完整冒险。** 当前步骤能够完成“选择—生成—下载”，但孩子为什么要选择、选择后修复了什么、下一步为何发生，仍缺少贯穿始终的故事因果。
2. **知识出现了，但探索过程不足。** 纹饰、传统色、用途等信息已有文案基础，仍需转化为观察、比较、试验和反馈，而不是只让孩子浏览标签后做选择。
3. **AI更接近生成工具，伙伴角色尚未建立。** AI目前主要承担提示词组装和图像生成，尚缺少提问、提示、陪伴、纠错边界与情绪回应机制。
4. **角色关系和差异需加强。** 镇水兽与吸水兽已有不同名称、姿态与位置描述，但需在任务、能力、性格、动作和文化解释中持续保持差异，避免最终只成为两套外形选项。
5. **数字成果到文化意义之间缺少仪式。** 2D、3D和打印链路已经存在，但“生成成功”还需要被叙事化为“记忆复原、使命重启和守护者诞生”。
6. **项目口径尚未完全统一。** 现有资料中存在名称、年龄层、页面步骤及神兽姿态描述不一致，需在后续章节开始前确认。
7. **文化内容缺少统一证据等级。** 部分纹饰寓意、年代、位置和神兽称谓已进入页面或配置，但尚未看到完整、逐条对应的权威资料来源表。

### 2.4 为什么需要情景式代入

儿童不会因为“这里有一份重要知识”就自然产生探索动力。情景式代入把抽象学习目标转换为孩子能够立即理解的行动：一只守护兽忘记了自己是谁，而我能通过发现线索帮助它醒来。

这种设定有四个作用：

- 给每次操作明确原因：清理泥土是为了看见线索，选择纹饰是为了修复缺失部分；
- 给知识建立记忆位置：用途与桥、水、城门的场景联系在一起，而不是孤立名词；
- 给AI建立合适边界：AI能辅助修复，却必须依靠孩子观察和判断；
- 给最终作品增加情感价值：孩子带走的不只是“生成的一只兽”，而是自己帮助唤醒的守护伙伴。

### 2.5 一句话项目定位

**一场以北京中轴线文化遗产为线索、由孩子与AI共同完成的文物记忆探索与数字文创冒险。**

### 2.6 产品形态

项目以响应式Web交互为核心载体，可用于个人浏览、亲子共玩、学校研学课堂、博物馆或遗产展示空间，并与以下形态衔接：

- 手绘中轴线地图与章节式文化探索；
- 触摸、拖拽、清理、对比、组合等轻交互任务；
- AI探索伙伴及分层提示；
- 个性化2D形象、数字收藏卡和神兽档案；
- 可旋转查看的3D模型及可打印文件；
- 3D打印手办、手机链、钥匙扣、冰箱贴等实体文创；
- 面向教师或讲解员的研学组织与成果分享材料。

### 2.7 核心价值

| 价值维度 | 核心价值 |
| --- | --- |
| 文化价值 | 让中轴线文化从“被讲述的知识”变成“可观察、可验证、可创造的经验” |
| 儿童体验 | 让孩子在悬念、反馈、陪伴和成就中主动完成探索 |
| 教育价值 | 训练观察、比较、推理、选择、表达和创造，而非只记忆结论 |
| AI素养 | 让孩子理解AI需要人的线索、判断与审美输入，生成结果也需要选择和检验 |
| 文创价值 | 把文化理解转化为有个人印记的数字作品与实体纪念物 |
| 扩展价值 | 以正阳门首章验证框架，再扩展到中轴线其他站点和守护角色 |

### 2.8 与普通文物介绍网站的区别

普通文物介绍网站通常以“展品—图片—说明文字”为主要结构，用户的核心行为是阅读与浏览。《中轴奇游记》的核心结构则是“悬念—观察—行动—反馈—复原—带走”：

- 孩子不是信息接收者，而是承担任务的探索者；
- 文物不是静态展项，而是等待找回线索的故事角色；
- 知识不是操作前的说明书，而是操作中发现和验证的依据；
- AI不是替孩子回答问题的老师，而是依赖孩子提供证据的修复伙伴；
- 最终结果不是看完一个页面，而是形成一只能够讲述其来源、选择与守护使命的个性化作品。

## 3. 核心设计理念与设计原则

### 3.1 核心设计理念

**让孩子通过亲手找回一件文物的记忆，与北京中轴线建立属于自己的连接。**

体验以“发现”为起点，以“修复”为过程，以“创造”为结果。文化信息、AI生成和文创转化都服务于同一件事：孩子先理解守护兽从哪里来、为何存在，再决定它如何以新的形象继续守护。

### 3.2 设计原则

#### 原则一：故事驱动，不做知识堆积

先让孩子理解任务、悬念与角色关系，再让知识作为解开问题的线索进入。每段知识都应回答当前任务中的一个问题，避免脱离情境的长篇讲解。

#### 原则二：探索优先，不以讲解代替发现

优先使用观察、触摸、擦除、拖拽、组合、对比和试验。必要讲解应短、及时、与刚才的操作直接相关，并允许孩子通过分层提示继续自主判断。

#### 原则三：AI是伙伴，不是老师

AI负责提出问题、整理线索、提供适量提示、鼓励尝试并执行数字复原；不直接公布全部答案，不替孩子完成关键选择，也不把生成结果包装成历史原貌。

#### 原则四：文化必须能够被体验

文物用途通过桥、水流与守护动作理解；纹饰通过局部观察、拓印、拼合或叠加理解；传统颜色通过材料感、环境变化与上色反馈理解。能用动作和反馈表达的内容，不只依赖文字。

#### 原则五：每次操作都产生可感知反馈

孩子完成动作后，应立即看到、听到或感到变化，例如泥土脱落、轮廓显现、纹饰发亮、色彩回流、眼睛睁开、水流恢复或记忆碎片归位。反馈既说明“操作成功”，也推进故事状态。

#### 原则六：知识可靠，创作边界清楚

史实、文化解读与故事设定分层管理。涉及地点、年代、用途、器物关系、纹饰和传统色的结论必须有可靠来源；拟人化和奇幻叙事可以增强体验，但不得伪装成历史事实。

#### 原则七：始终从儿童视角检验

每个页面都应能让孩子迅速回答：“我要做什么”“为什么要做”“做完发生什么”。控制单屏信息量和任务长度，减少成人化术语；让失败成为获得新线索的过程，而不是惩罚。

#### 原则八：保留选择的意义

孩子的纹饰、颜色、姿态、表情和附加元素选择应真实影响视觉结果、角色档案或后续反馈。文化事实不由投票改变，但个性化创作应有足够空间，不设置唯一的“最好看答案”。

#### 原则九：维持角色与文化原型差异

镇水兽与吸水兽是不同的守护形象。后续策划必须分别保持其姿态、所处环境、功能线索、行为反馈和性格表达，不以通用“萌兽模板”覆盖差异。

#### 原则十：形成数字体验到实体文创的完整旅程

数字守护兽、2D形象、3D模型、收藏卡和实体产品应共享同一份“记忆档案”，保留孩子的选择、文化出处与创作说明。实体化不是附加下载按钮，而是冒险完成后“把守护使命带回生活”的最后一程。
