### 2.2 JS 加载顺序（有依赖关系，不可打乱）

```html
1. config.js          ← 最先加载，定义全局变量和配置
2. core.js            ← 依赖 config.js
3. ai-generate.js     ← 依赖 config.js, core.js
4. meshy-3d.js        ← 依赖 config.js, core.js
5. share-integration.js ← 依赖 meshy-3d.js（state 变量）⚡新增
6. interactions.js    ← 主入口，依赖以上所有
7. patrol-game.js     ← 独立模块
8. home-story-config.js ← 独立配置
9. home-story.js      ← 依赖 core.js
```

> ⚠️ 朋友修改时注意：share-integration.js 是 7月16日新增的，依赖 meshy-3d.js 中的 `state` 变量。

## 三、3D 分享系统（新架构，7月16日上线）

### 3.1 数据流

```
学生操作研学网站（mindbubble.cloud）
│
├─ 1. AI 生成 2D 神兽图（Seedream API via Cloudflare Worker）
├─ 2. Meshy 生成 3D 模型（Meshy API via Cloudflare Worker）
├─ 3. 获得 Meshy 下载 URL（STL/3MF 格式）
│
├─ 4. 点击「分享给朋友看」按钮
│   └─ 前端把 Meshy URL 发送给后端（不经过浏览器下载 STL）
│
├─ 5. 后端（share.mindbubble.cloud）
│   ├─ 从 Meshy CDN 下载 STL 文件
│   ├─ 自动修复模型（stl_repair.py）
│   ├─ 保存修复后的文件
│   └─ 返回分享链接
│
└─ 6. 分享链接 → H5 页面
    ├─ Three.js 渲染 3D 模型（可旋转、缩放）
    ├─ 展示学生姓名、作品名、AI 原画
    └─ 提供 STL 下载按钮
```

### 3.2 后端 API 列表

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/work/upload` | POST | 上传作品（接收 model_url 参数） |
| `/api/work/detail` | GET | 获取作品详情 |
| `/api/work/download` | GET | 下载 STL 文件 |
| `/api/work/stl` | GET | 获取 STL 供 Three.js 预览 |
| `/share` | GET | H5 分享页面 |

## 十、快速排错指南

### 前端页面不更新？
1. 确认代码已 push 到 main 分支
2. 等 2-3 分钟让 Cloudflare Pages 自动部署
3. 浏览器强制刷新：Ctrl+Shift+R 或无痕窗口

### 分享功能失败？
1. 检查后端服务：服务器 Web 终端执行 `systemctl status ai-magic-share`
2. 检查日志：`journalctl -u ai-magic-share --since "10 min ago" --no-pager`
3. 浏览器无痕窗口测试（排除缓存问题）

### Meshy 3D 生成失败？
1. Meshy API 有限流，出现"请求太频繁"需等待几分钟后重试
2. 可使用"使用预存AI图片"按钮跳过 AI 生成
