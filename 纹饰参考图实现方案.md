# 纹饰参考图实现方案

## 目标
让用户在页面上选择的纹饰真正影响 AI 生成的 2D 图片效果，而不只是文字描述。

## 核心发现
Worker 代理（`meshy-proxy.js`）是**纯透传**——前端请求体原封不动转发给 Seedream API。因此：
- ✅ **Worker 不需要任何改动**
- ✅ 只需修改前端代码 + 准备纹饰参考图素材

---

## 实施步骤

### 第一步：准备 8 张纹饰参考图

为 config.js 中的 8 种纹饰各准备一张清晰的参考图：

| ID | 纹饰名称 | 参考图要求 |
|----|---------|-----------|
| limb | 肢部纹饰 | 层叠水鳞纹理，类似龙鳞/鱼鳞的重复图案 |
| mouth_horn | 嘴角纹饰 | 火焰纹/卷草纹，嘴角两侧的装饰纹样 |
| mane | 鬃毛 | 飘逸的鬃毛，带水波纹流动感 |
| ear | 耳部纹饰 | 旋涡状纹样，类似水涡 |
| eye | 眼部纹饰 | 圆目造型，突出眼神的装饰纹 |
| head_horn | 头部倚角 | 鹿角/龙角造型，角上带纹饰 |
| back_cloud | 脊背云纹 | 中国传统祥云纹，如意云头 |
| neck_mane | 颈部鬃髯 | 长须/鬃髯，飘带感 |

**图片规格要求：**
- 尺寸：512×512 或 1024×1024（正方形，纹饰居中）
- 格式：PNG（透明背景）或 JPG（白色/干净背景）
- 风格：中国传统纹样，清晰可辨
- 存储：上传到腾讯云 COS（`assets/patterns/` 目录）

**图片来源方案（三选一）：**
- **A. AI 生成**：用 Seedream 生成 8 张传统纹样图，人工挑选（推荐，风格统一）
- **B. 公开素材**：从故宫博物院数字文物库、中国传统纹样素材库等获取
- **C. 手绘扫描**：课程现场带小朋友手绘后扫描（最有教学意义，但需线下完成）

**建议先用方案 A 快速出效果，后续可替换为方案 C 增加教学互动感。**

```javascript
// 改前
```

每种纹饰都加一个 `image` 字段，指向对应的参考图 URL。

### 第三步：修改 ai-generate.js — 传递参考图给 API

#### 3.1 改造 `callSeedreamAPI` 函数

```javascript
// 改前：只传 prompt
async function callSeedreamAPI(prompt) {
    // ...
    body: JSON.stringify({
        model: AI_CONFIG.model,
        prompt: prompt,
        size: AI_CONFIG.size,
        response_format: 'url',
        watermark: false
    })
}

// 改后：支持传入参考图数组
async function callSeedreamAPI(prompt, options) {
    options = options || {};
    var requestBody = {
        model: AI_CONFIG.model,
        prompt: prompt,
        size: AI_CONFIG.size,
        response_format: 'url',
        watermark: false
    };
    // 如果有参考图，加入 image 字段
    if (options.refImages && options.refImages.length > 0) {
        requestBody.image = options.refImages;  // Seedream 4.5 支持图片URL数组
    }
    // ...
    body: JSON.stringify(requestBody)
}
```

#### 3.2 改造 `generateAICandidatesGeneric` 函数

```javascript
// 改前
async function generateAICandidatesGeneric(grid, basePrompt, stylesConfig) {
    // ...
    var fullPrompt = basePrompt + style.suffix;
    return callSeedreamAPI(fullPrompt).then(...)
}

// 改后：增加 refImages 参数
async function generateAICandidatesGeneric(grid, basePrompt, stylesConfig, refImages) {
    // ...
    var fullPrompt = basePrompt + style.suffix;
    return callSeedreamAPI(fullPrompt, { refImages: refImages }).then(...)
}
```

#### 3.3 改造 `generateCandidates` 函数 — 收集纹饰参考图

在神兽模式的分支中，收集用户选中的纹饰参考图：

```javascript
// 在 generateCandidates() 的神兽分支中
} else {
    // 收集选中纹饰的参考图 URL
    var patternRefImages = [];
    if (state.selectedPatterns && state.selectedPatterns.length > 0) {
        state.selectedPatterns.forEach(function(p) {
            var pt = patterns.find(function(x) { return x.id === p; });
            if (pt && pt.image) patternRefImages.push(pt.image);
        });
    }
    
    var beastStyles = [ /* 不变 */ ];
    await generateAICandidatesGeneric(grid, basePrompt, beastStyles, patternRefImages);
}
```

## 改动文件清单

| 文件 | 改动内容 | 工作量 |
|------|---------|--------|
| `js/config.js` | 8 个 pattern 对象各加 `image` 字段 | 小 |
| `js/ai-generate.js` | 改造 3 个函数签名 + 收集参考图逻辑 | 中 |
| Worker `meshy-proxy.js` | **不需要改动** | 无 |
| 腾讯云 COS | 上传 8 张纹饰参考图到 `assets/patterns/` | 需准备素材 |

## 预计效果

- 用户选择的纹饰会通过 Seedream 4.5 的 `image` 参考图能力，直接影响生成结果
- 最多可选 8 张纹饰参考图（Seedream 4.5 支持最多 14 张参考图）
- 生成的神兽图片应能体现出类似选中纹饰的纹理、造型特征

## 风险与降级

- 如果参考图效果不理想，可以保留 `image` 字段但传空数组，退化为纯 prompt 模式
- 如果某些纹饰图片与神兽风格冲突，可在 prompt 中增加"纹样作为身体装饰纹理融入"等约束词
